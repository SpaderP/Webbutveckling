!DOCTYPE

<DOCTYPE html> är ett sätt för webbläsaren att veta att vilken version av Html som dokumetet använder sig av.


Teckenkod - : <html lang="sv">

Teckenkod är en standard som talar om för datorn att den ska konvertera texten till en sekvens av bytes när den sparas i en fil. Då det finns många olika typer av tecken i världen. I dagsläget används UTF-8 som är den snabb och kompakt och innehåller alla icke engelska tecken. Den talat om för webläsaren och datorn hur man ska uppföra sig när den ser ett antal tecken.


Språk

Detta görs helt enkelt så här: <html lang="en">. Det attribut vi lade till är då lang som då anger vilket språk som sidan är skriven i. I detta fall är det engelska som är språket. Detta är ett språk attrubut som förenklar för sökmotorer vid sökning vilket filtrerear ollika språk vid sökning. Har man exempelvis detta hjälper då text textläsare att bestämma vilket språk den ska läsa upp texten på

Validering

Genom en validator undersöks din kod så att den uppfyller de standarder som finns. Genom en validator kan man kolla om ens kod uppfyller de konventionella standarder på en hemsida. Några exempel på vad för För att börja göra en hemsida krävs det vissa koder.


<!DOCTYPE html>
<html lang="sv">
<head>
<meta charset="utf-8">
<title>A Tiny HTML Document</title>
<link href="styles.css" rel="stylesheet">
<script src="scripts.js"></script>
</head>
<body>
<p>Hello world.</p>
</body>
</html>

Struktur

html <html></html> Detta referarar till själva huvudroten i html dokumenetet. <html></html> är taggarna för all kontenta som finns inom dessa ramar. Förutom <!DOCTYPE>

head Detta rfererar till alla metadata som finns i dokumentet

body Detta representerar huvudkontentan i dokumentet eller hemsidan.

title Definerar titeln på dokumentet alltså hemsidan. Vilket står på webbläsaren.


Metainformation:

Definerar titeln på dokumentet alltså hemsidan. Vilket står på webbläsaren.

Meta-taggar med namn attribut: Definerar titeln på dokumentet alltså hemsidan. Vilket står på webbläsaren.
Author Bestämmer namnet på sidans författare.
Description Förklarar sidans kontenta.
Generator Identifierar mjukvaran som generar dokumentet.
Keywords Bestämmer namnet på sidans författare.
Robots Förklarar sidans kontenta. Det finns vissa tags som inte är lika enkla att förstå och bland dessa finns det t.ex. robots, generator och http-equiv. Robots används för att ange information för de botar (kallas ibland spindlar eller robots) som indexerar för sökmotorer. Man kan ange om sidan skall bli indexerad och om länkar på sidan skall följas eller inte.


Meta-tabbar med attributer:

http-equiv
när http_equiv attributen är specifierad på meta elemnetet då blir det ett pragmatisk direktiv. Http-equiv tag skall inte användas tillsammans med name tagen. Http-equiv simulerar något som kallas HTTP response header. Detta är den information som en webserver får efter att den gjort en request. Eftersom man inte alltid har tillåtelse att konfigurera den webserver man använder kan man använda http-equiv för att som sagt simulera vad för respons man vill skicka till webservern. Denna information ges då allmänt inom header elementet men http-equiv låter en göra lite extra. Man kan t.ex. säga att en sida skall laddas om vid ett visst intervall. Varjde sida bör ange vilken teckenkodning som används och det görs genom en tag kallad charset. Denna sida använder t.ex. utf-8. Detta anges på följande sätt: <meta charset="utf-8">. Head används också till att länka till filer med CSS eller JavaScript kod. Man kan också lägga in CSS kod via <style> element och JavaScript kod via <script> element.
